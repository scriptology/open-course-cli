# Open Course CLI

A terminal AI tutor for language learning. It runs in the terminal; exercises, lessons, and answer analysis are generated by a chosen LLM provider.

## Running locally

Requires Rust and cargo.

```bash
cd open-course-cli
cargo run
```

By default data is stored in the current directory under `.open-course-cli/`. To use a different directory:

```bash
cargo run -- --data-dir /path/to/project
```

Quit the app with `Ctrl+C` or `Esc`/`q` depending on the screen.

## First run (onboarding)

A setup wizard appears on first launch. Move between fields with:

- `Tab` / `Enter` — next field
- `Shift+Tab` / `↑` — previous field
- `Esc` — quit

Fields:

1. **Native language** — two-letter ISO 639-1 code, e.g. `ru`, `en`.
2. **Target language** — language you are learning, e.g. `en`, `es`, `de`.
3. **Age** — optional. Used to pick age-appropriate contexts for exercises (for example, to avoid school or kindergarten scenarios for adult learners).
4. **CEFR** — your self-assessed level: `A1`, `A2`, `B1`, `B2`, `C1`, `C2` (required).
5. **Batch size** — number of exercises per session: `2`, `3`, `4`, `5`.
6. **Provider** — `openai`, `anthropic`, `google`, `deepseek`, `mistral`, `openrouter`, `ollama`, or `custom`. Can be changed later in settings.
7. **API key** — provider API key. Can be left empty for Ollama/Custom. Entered keys are masked (`********`).
8. **Base URL** — shown for `custom` and `ollama`:
   - `custom` — required, default `https://opencode.ai/zen/go/v1`.
   - `ollama` — optional, default `http://localhost:11434/v1`.
   - Skipped for other providers.
9. **Endpoint** — shown for `custom`:
   - `chat/completions` — OpenAI-compatible endpoints (Ollama, OpenRouter, OpenCode Go DeepSeek/GLM/Kimi/MiMo).
   - `messages` — Anthropic Messages API (Anthropic, OpenCode Go Qwen/MiniMax).
   - Built-in providers set the endpoint automatically.
10. **Model** — model picker. The app fetches available models from the provider:
    - `↑`/`↓` — select model.
    - `Enter` — confirm selection.
    - `m` — switch to manual model ID input.
    - `r` — retry fetching the model list.

The final step saves `config.json` to `.open-course-cli/`, runs model diagnostics, and opens the **Curriculum** view so you can generate the first course.

## OpenCode Go

OpenCode Go uses two endpoints:

- `https://opencode.ai/zen/go/v1/chat/completions` — GLM, Kimi, DeepSeek, MiMo.
- `https://opencode.ai/zen/go/v1/messages` — Qwen, MiniMax.

In settings set Base URL to `https://opencode.ai/zen/go/v1`, then on the **Endpoint** step choose `chat/completions` or `messages` depending on the model. The app automatically uses the right client.

## Where data is stored

The `--data-dir` directory gets a `.open-course-cli/` folder:

- `config.json` — profile, provider, and preferences.
- `db/` — LanceDB: progress, session history, curriculum topics, reviews, and topic docs.
- `config.json.tmp` — temporary file; safe to delete if the app crashed.

## Screen commands

### Global

| Key | Action |
|-----|----------|
| `Ctrl+C` | Quit the app |
| `Esc` | Back / cancel / quit |

### Dashboard

| Key | Action |
|-----|----------|
| `n` | New session |
| `r` | Topic review |
| `d` | Topic documentation |
| `c` | Curriculum list |
| `p` | Language pairs |
| `s` | Settings |
| `q` | Quit |

### Session

First pick a topic, then solve the exercises.

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Select topic |
| `Enter` | Start session for the selected topic |
| `Esc` | Cancel and return to the dashboard |
| `Backspace` | Delete a character of the answer |
| `Enter` | Submit translation and go to the next exercise |

After the last exercise the answers are sent to the LLM for analysis. While generation/analysis is in progress a spinner is shown. `Esc` cancels the request.

### Session report

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Scroll the report |
| `n` | New session |
| `r` | Review |
| `w` | Practice a weak topic |
| `d` | Documentation |
| `Esc` | Dashboard |

### Topic review

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Select topic |
| `s` | Toggle sort |
| `Enter` | Start review for the selected topic |
| `Esc` | Back |

### Topic docs

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Select topic in the list |
| `Enter` | Open the topic article |
| `e` | Regenerate the article |
| `p` | Practice this topic (start a session) |
| `↑`/`↓` or `j`/`k` | Scroll the article |
| `Esc` | Back to the list / dashboard |

### Curriculum

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Select topic |
| `g` | Generate the full curriculum |
| `r` | Reset curriculum, progress, and reviews, then regenerate (only when non-empty) |
| `a` | Add 5 new topics extending the existing curriculum (only when non-empty) |
| `s` | Toggle sort: progression vs score (only when non-empty) |
| `m` | Change the current model (opens Settings > Provider > Model) |
| `Enter` | Open the selected topic in Docs |
| `Esc` | Back to Dashboard |

### Language pairs

| Key | Action |
|-----|----------|
| `↑`/`↓` or `j`/`k` | Select pair |
| `Enter` | Switch to the selected pair |
| `a` | Add a new language pair (mini-onboarding: native/target/age/CEFR) |
| `Esc` | Back to Dashboard |

Each pair has its own isolated database folder under `.open-course-cli/pairs/{native-target}/db`. Progress, history, curriculum, and reviews are not shared between pairs. Provider settings and preferences are global.

### Settings

| Key | Action |
|-----|----------|
| `Tab` / `Shift+Tab` | Next / previous field |
| `←` / `→` | Switch section (Provider / Profile / Session / Data) |
| `Enter` | Save the current field |
| `↑` / `↓` | Select provider when the Provider field is active |
| `Esc` | Return to the dashboard |

Settings sections:

- **Provider** — provider, base URL, endpoint, API key, model.
- **Profile** — age and CEFR for the active language pair. To change the native/target languages, add a new pair from the dashboard (`p`).
- **Session** — batch size (2–5), hint mode (`auto` / `on-demand`).
- **Data** — reset progress, history, curriculum, reviews, or everything. Requires `y` confirmation.

## Architecture

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────┐
│   TUI screens   │────▶│   AppState   │────▶│   LanceDB   │
│ (dashboard,     │     │              │     │             │
│  session, docs) │◀────│              │◀────│             │
└─────────────────┘     └──────────────┘     └─────────────┘
                               │
                               ▼
                        ┌──────────────┐
                        │  rig-core    │
                        │  LLM client  │
                        └──────────────┘
```

Flow:

1. User selects an action in the TUI.
2. If an LLM call is needed, `AppState` spawns a `tokio` task and shows a spinner.
3. The LLM result arrives on a channel and is rendered on screen.
4. Session, progress, and generated materials are saved to LanceDB.

## Manual smoke test

Minimal manual scenario:

1. Run `cargo run` in an empty directory.
2. Complete onboarding: provider `ollama`, pick a model from the list (or type one manually, e.g. `llama3`), languages `ru` → `en`.
3. On the dashboard press `c` to open the curriculum, then `g` to generate the course if needed.
4. Select a topic and press `Enter` to open its docs.
5. Press `p` in docs to start a session for that topic.
6. Solve the exercises and wait for the report.
7. Press `d` in the report to open docs for a weak topic.
8. Open `s` (Settings) and verify that provider/model list loading with `l` works.

If the LLM is unavailable, use `--data-dir` pointing to an existing `.open-course-cli/` folder with a saved curriculum and progress.
